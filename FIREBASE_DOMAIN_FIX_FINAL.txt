ERROR:
------
"A valid domain name is required (e.g. 'myapp.com')"

CAUSE:
------
Firebase doesn't accept localhost:5000 - it requires a real domain name.

SOLUTION:
---------
Use ngrok to create a real domain that tunnels to localhost:5000

WHAT IS NGROK?
--------------
A tool that creates a public URL for your local app.
Example: https://abc123.ngrok.io → http://localhost:5000

QUICK FIX (5 MINUTES):
----------------------

1. Install ngrok
   Command: brew install ngrok

2. Start ngrok
   Command: ngrok http 5000

3. Copy the URL
   You'll see: https://abc123.ngrok.io
   Copy: abc123.ngrok.io

4. Add to Firebase
   - Go to: https://console.firebase.google.com/
   - Select: unvibe-54ae1
   - Authentication → Settings
   - Authorized domains → Add domain
   - Type: abc123.ngrok.io
   - Click: Add

5. Wait 5-10 seconds

6. Test
   - Go to: https://abc123.ngrok.io (use HTTPS!)
   - Click: Sign in with Google
   - Should work ✅

IMPORTANT:
----------
✅ Use HTTPS (not HTTP)
✅ Copy only domain (abc123.ngrok.io)
✅ Don't include https:// when adding to Firebase
✅ Wait 5-10 seconds after adding
✅ Clear cache if needed (Cmd+Shift+Delete)
